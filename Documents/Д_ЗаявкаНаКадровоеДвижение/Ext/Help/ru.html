Введение.

Служебные записки в электронном документообороте 1С – это электронная версия внутренней служебной документации, которая может содержать в себе внутренние приказы, корректировки бюджета, оповещения, согласования, прочую служебную информацию.

Служебные записки в 1С могут быть 2-х типов:

- СЗ на корректировку бюджета;

- Произвольная СЗ.

 

1.       Типы СЗ.

1.1.  СЗ на корректировку бюджета.

Данный тип служебных записок используется в случае корректировки утвержденного бюджета текущего месяца. Имеет следующий вид (рис.1).

Рис.1

Порядок заполнения СЗ на корректировку бюджета следующий.

1)             Заполняется шапка служебной записки:

- поле «Дата». Служебка оформляется всегда текущей датой, поэтому поле «дата» недоступно.

- поле «Предприятие». Указывается предприятие/проект от имени которого оформляется служебная записка. Содержит список только доступных пользователю предприятий.

- поле «Вид СЗ». Корректировка бюджета.

- поле «Кому». Главный адресат служебной записки.

- поле «Тема». Тема СЗ.

- поле «Прикрепить файл». Возможно прикрепление предварительно загруженного в 1С файла любого формата. Размер файла не должен превышать 2 Мбайта.

2)             Далее заполняется табличная часть служебной записки:

- поле «ЦФО (проект)». ЦФО - Центр финансовой ответственности. Автор может корректировать лимиты не только доступных ему предприятий, но и лимиты любого другого предприятия/проекта Компании.

- поле «Статья ДДС». Указывается статья бюджета Движения Денежных Средств.

- после указания «Статьи ДДС» можно использовать автоматическое получение текущих лимитов по данной статье. Для этого используется кнопка «Получить текущий лимит по статье». После нажатия данной кнопки поля «Лимиты на начало затраты» и «Лимит на начало ДДС» должны заполнится автоматически. Если этого не произошло, есть возможность указать лимиты вручную.

«Лимиты на затраты» - это лимит на начисления (производственная себестоимость или бюджет доходов и расходов (Финансовый результат)).

«Лимит на ДДС» - лимит на финансирование (ден. средства).

- поле «Сумма корректировки БДДС». Указывается сумма, на которую изменяется выбранная статья. В случае уменьшения сумма ставится с минусом.

Переброс лимита с одной статьи на другую соответственно оформляется двумя строками. В первой строке указывается статья на которую нужно перенести лимит, во второй строке статья за счет которой происходит перенос лимита.

- поле «Статья затрат». В большинстве случаев проставляется автоматически при выборе «Статьи ДДС». Если при выборе статьи поле не заполнилось, нужно указать статью затрат вручную.

- поле «Сумма корректировки затрат». Если корректировка ДДС несет за собой изменение себестоимость или финреза, то в данном поле указывается сумма корректировки. Сумма может быть отлична от Суммы корректировки БДДС.

- поле «Номенклатура» заполняется только в случае корректировок в себестоимость и в случае необходимости корректировки себестоимости конкретной номенклатуры.

- поле «Основание». Причина корректировки.

Порядок заполнения произвольной СЗ следующий.

3)             Заполняется шапка служебной записки:

- поле «Дата». Служебка оформляется всегда текущей датой, поэтому поле «дата» недоступно.

- поле «Предприятие». Указывается предприятие/проект от имени которого оформляется служебная записка. Содержит список только доступных пользователю предприятий.

- поле «Вид СЗ». Произвольная.

- галочка «Доступна пользователям предприятия». Галочка определяет доступность произвольной служебной записки пользователям, для которых доступно предприятие, от имени которого пишется служебная записка. В данном примере все пользователи, которым доступно предприятие А увидят эту СЗ в своих списках. И наоборот если галочка не установлена, то служебку видят только автор, «согласователи» и адресаты.

- поле «Кому». Главный адресат служебной записки (в случае произвольной СЗ их может быть несколько, добавляются при помощи кнопки ).

- переключатель «По очереди/Всем сразу». Если выбрано «По очереди», то утверждение СЗ будет происходить в порядке очередности. Задача «Утвердить СЗ» будет создаваться для каждого следующего пользователя в маршруте утверждения. И если один из пользователей отправит СЗ на доработку, то она вернется автору. Если выбрано «Всем сразу», то задачи «Утвердить СЗ» создаются для всех пользователей, указанных в маршруте, одновременно. Если один из пользователей не утвердит СЗ, то она вернется автору на доработку после того как все выполнят задачи «Утвердить СЗ». Если утвердят все, то служебка пойдет далее по маршруту.

- поле «Тема». Тема СЗ.

-  вкладка «Табличные данные». В произвольную СЗ могут быть добавлены табличные данные (есть возможность копирования из Excel). Вкладка доступна, если стоит галочка «Добавить таблицу».

- поле «Прикрепить файл». Возможно прикрепление предварительно загруженного в 1С файла любого формата. Размер файла не должен превышать 2 Мбайта.

 

2.       Запуск процесса утверждения СЗ (вне зависимости от вида СЗ запуск утверждения одинаков).

 

После заполнения полей служебной записки можно переходить к запуску процесса утверждения. Для этого нужно нажать кнопку «Стартовать процесс утверждения». Откроется окно (рис.3), в котором вводятся все параметры процесса утверждения.

Рис.3

 

 

На вкладке «Согласование» указывается маршрут согласования служебки. Табличная часть согласования может быть частично заполнена данными служебки из поля «Кому».

На вкладке «Ознакомление» указывается перечень пользователей, которые будут оповещены об утверждении служебной записки.

После заполнения данной формы необходимо нажать на кнопку «Стартовать и закрыть». Процесс утверждения служебной записки будет запущен. Служебная записка будет защищена от редактирования.

 

3.       Стадии утверждения СЗ.

В процессе утверждения, СЗ может находится на следующих стадиях:

1)             Отправка СЗ (рис.4). Задача отправки может возникнуть только у автора СЗ.

Рис. 4

 

В новом варианте процесса данная стадия возможно только если служебка отправлена на доработку. В форме доработки можно отредактировать маршрут утверждения на вкладке «Согласование» (но только несогласовавших пользователей), перечень пользователей, которых нужно ознакомить при утверждении СЗ на вкладке «Ознакомление». Данное редактирование не повлечет за собой отмену решения ранее пользователей уже согласовавших СЗ.

В случае если нужно скорректировать содержание служебки, нужно сначала снять защиту на редактирование СЗ. Для этого существует кнопка . Однако при снятии запрета на редактирование все пользователи, ранее согласовавшие данную СЗ, должны будут заново ее согласовать.

Кнопка «Ознакомится и завершить БП» используется в случае если служебная записка в данном виде не моет быть утверждена и означает завершение процесса утверждения с формулировкой «СЗ не утверждена».

 

2)             Поверка СЗ, Утверждение СЗ, Ознакомление с результатами СЗ (рис.5)

Задачи проверки, утверждения и ознакомления похожи. Форма задачи имеет кнопки «Утвердить» и «На доработку». Также имеются дополнительные кнопки печати.

На вкладке «Согласование» можно просмотреть текущее состояние маршрута СЗ, а также отредактировать его (кроме случаев отправки СЗ сразу всем).

Рис.5

 

 

Кроме этого присутствует ссылка на СЗ, история переписки с комментариями согласователей и поле «Комментарий», содержание которого поможет автору СЗ увидеть причину в случае отказа утверждения СЗ или доп. комментарии согласователей.

В случае нажатия кнопки «Утвердить», СЗ пройдет далее по маршруту, в случае нажатия «На доработку» СЗ вернется автору. Желательно указывать причину отказа в комментариях к задаче.

Задача «Ознакомление с результатами СЗ» возникает у автора и пользователей, от которых ожидается подтверждение ознакомления, в случае прохождения стадии утверждения и проверки.

В случае положительного результата утверждения, будет автоматически скорректирован текущий лимит по указанным статьям. Скорректированный лимит будет использован в отчете по ДДС и заявках на оплату.

Помимо этого в форме списка можно использовать дополнительные команды:

- «Вернуть на доработку». Возможен принудительный возврат служебной записки на доработку, который можно выполнить на всех стадиях кроме последней стадии исполнения.

- «Изменить маршрут». При нажатии данной кнопки откроется форма изменения маршрута служебки (рис 7.). В данной форме можно изменить субъекты согласования, но только для пользователей еще не согласовавших служебку. Также можно добавить новых пользователей в маршрут утверждения. На рисунке 3 в первой строчке пользователь «Руководитель ФС» будет заменен в маршруте на пользователя «Зам. Руководителя ФС», а пользователь «Директор Д» будет добавлен в маршрут согласования.

Рис.7

- «Предоставить доступ пользователям к СЗ». При нажатии откроется форма (рис.8), в которой можно указать пользователей, которые будут видеть указанную СЗ в своем списке служебок. Если галочка «Создавать оповещение для пользователей» установлена, то указанные пользователи получат сообщение на рабочий стол о доступности служебки.

Рис.8

Доступны команды печати (открывают печатные формы, которые затем могут быть распечатаны). (рис.9)

Рис.9

Рис.10

 

Команда «Отчет по выполнению» покажет стадии прохождения процесса утверждения с временными интервалами. (рис. 11)
